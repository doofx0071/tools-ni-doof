## ðŸ“‚ Modular Tool Architecture
Each tool defines its own universe. Maintain strict isolation:

### 1. Automation First
- **Always** use `bun make-tool [name]` to start a new tool.
- Follow the interactive prompts to select a **TweakCN Theme** (e.g., Northern Lights, Cyberpunk).
- This automatically generates the `tool.css`, folders, and routing.

### 2. Folder Structure
- **Frontend Logic**: `src/features/[tool-name]/`
  - `components/`: Tool-specific UI components (e.g., `DashboardView.tsx`, `SettingsView.tsx`).
  - `hooks/`: Custom React hooks.
  - `styles/tool.css`: Scoped CSS variables (e.g., `primary`, `accent`) generated by the CLI.
  - `utils/`: Helper functions specific to this tool.
  - `types/`: TypeScript definitions.
  - `index.tsx`: The main entry point for the tool.
- **Backend Logic**: `convex/[tool_name]/` (Must use underscores, not hyphens)
  - `queries.ts`: For reading data (reactive).
  - `mutations.ts`: For modifying data.
  - `schema.ts`: Define table structures in `convex/schema.ts` (centralized).

## ðŸ”Œ Convex Backend Rules
1. **Schema Definitions**: Always define tables in `convex/schema.ts` using `defineTable`.
2. **Type Safety**: Use `v` from `convex/values` to validate arguments and schema fields.
3. **Separation of Concerns**:
   - **Queries**: fetching data only.
   - **Mutations**: writing data only.
   - **Actions**: performing external API calls (fetch, etc).
4. **Context usage**: Use `ctx.db` for database operations and `ctx.auth` for user context.

## âœ¨ UI/UX Principles
- **Scoped Styling**: Always wrap your tool's main container in the scoped class (e.g., `.tool-webhook-tester`) to apply the TweakCN theme.
- **Glassmorphism**: Use the `.glass` utility for cards and panels.
- **Micro-animations**: Every interactive element should have a smooth Framer Motion transition.

## ðŸ“± Mobile & Responsiveness Rules (CRITICAL)
- **Mobile-First IS MANDATORY**: Always build for mobile screens (320px+) first, then scale up using `md:` and `lg:` prefixes.
- **No Horizontal Scroll**: Ensure main containers have `overflow-x-hidden`.
- **Touch-Friendly**: All interactive elements (buttons, inputs) must have a minimum height of **44px** for touch targets.
- **Responsive Grids**:
  - Mobile: `grid-cols-1` or `flex-col`
  - Tablet: `grid-cols-2`
  - Desktop: `grid-cols-3` or `grid-cols-4`
- **Component Scaling**:
  - Use `w-full` on mobile for inputs and cards.
  - Stack sidebars/panels vertically on mobile.
  - Use `Drawer` or `Sheet` for mobile menus instead of dense headers.
- **Interactive Elements**: ALL buttons, links, and clickable elements must have `cursor-pointer` and a hover state (e.g., opacity change or scale).
- **Testing**: You must verify layout logic for `sm` (640px) and `md` (768px) breakpoints.
